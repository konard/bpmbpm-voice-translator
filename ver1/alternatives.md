## Детальное описание альтернатив

### 1. Выбранный подход: Browser Web Speech API + MyMemory API

**Распознавание речи:** `SpeechRecognition` — стандартный браузерный API.

**Перевод:** [MyMemory](https://mymemory.translated.net/) — бесплатный REST API перевода без регистрации.

**Синтез речи:** `SpeechSynthesis` — стандартный браузерный API.

**Плюсы:**
- Полностью браузерное, не нужен сервер
- Бесплатно, без регистрации
- Минимум зависимостей
- Работает на GitHub Pages

**Минусы:**
- `SpeechRecognition` только Chrome/Edge
- MyMemory: лимит 5000 символов/день без ключа
- Качество перевода ниже, чем у платных API

---

### 2. OpenAI Whisper + GPT-4o

**Распознавание речи:** Запись через `MediaRecorder`, отправка аудио в Whisper API.

**Перевод:** GPT-4o с промптом на перевод.

**Плюсы:**
- Очень высокое качество STT и перевода
- Whisper хорошо обрабатывает шум, акцент, смешанный язык

**Минусы:**
- Требует платный API-ключ OpenAI
- Высокая латентность (2–5 секунд на цикл)
- Хранение ключа в браузерном JS небезопасно

---

### 3. Google Cloud Speech-to-Text + Google Translate API

**Распознавание речи:** Google Cloud STT — потоковое распознавание через WebSocket.

**Перевод:** Google Translate API v2.

**Плюсы:**
- Лучшее качество потокового STT
- Поддержка >100 языков
- Официальная поддержка русского языка

**Минусы:**
- Платно (хотя есть бесплатный уровень)
- Требует API-ключ
- Необходим backend-прокси для скрытия ключа
- Нельзя напрямую с GitHub Pages без прокси

---

### 4. LibreTranslate (self-hosted)

**Перевод:** Открытая альтернатива Google Translate, можно развернуть локально.

**Плюсы:**
- Полностью открытый исходный код
- Высокая конфиденциальность данных
- Бесплатно

**Минусы:**
- Требует запуск сервера (Docker)
- Качество перевода ниже коммерческих решений
- Не работает с GitHub Pages без прокси

---

### 5. DeepL API (Free tier)

**Перевод:** DeepL — один из лучших переводчиков для европейских языков.

**Плюсы:**
- Высокое качество перевода, особенно для ru↔en
- Есть бесплатный уровень (500 000 символов/месяц)

**Минусы:**
- Требует регистрацию и API-ключ
- Хранение ключа в JS небезопасно без прокси
- Freemium — при превышении лимита платно

---

### 6. Подход на основе Web Audio API + VAD (Voice Activity Detection)

Вместо `SpeechRecognition` — собственная обработка аудиопотока с алгоритмом обнаружения голосовой активности (VAD).

**Плюсы:**
- Кроссбраузерность (Firefox, Safari)
- Более точное определение пауз

**Минусы:**
- Значительно сложнее в реализации
- Требует внешней библиотеки VAD или Wasm-модуля
- Всё равно нужен внешний STT API

## Наиболее проблемные моменты и ограничения

### Критические ограничения

1. **Браузерная совместимость SpeechRecognition**
   - Доступен только в Chrome и Edge (Chromium-based)
   - Firefox и Safari не поддерживают без флагов
   - **Митигация:** Показывать предупреждение пользователю при неподдерживаемом браузере

2. **Безопасность API-ключей**
   - Любой ключ, встроенный в браузерный JS, виден пользователю
   - **Митигация:** Использовать только бесплатные API без ключей (MyMemory) или иметь backend-прокси

3. **Определение конца блока речи**
   - `SpeechRecognition` не гарантирует точный момент конца фразы
   - Таймер паузы — эвристика, зависящая от скорости речи
   - **Митигация:** Настраиваемый параметр `pauseDuration` в конфигурации

4. **Лимиты бесплатного перевода**
   - MyMemory: 5000 символов/день анонимно
   - При интенсивном использовании лимит исчерпывается быстро
   - **Митигация:** Показывать ошибку с предложением использовать API-ключ MyMemory

### Менее критичные ограничения

5. **Качество голосов SpeechSynthesis**
   - Зависит от ОС и браузера
   - На Windows — системные голоса (часто роботизированные)
   - На macOS/iOS — более качественные голоса

6. **Задержка перевода**
   - Сетевой запрос к MyMemory занимает 300–1500 мс
   - Это приемлемо, но заметно при коротких фразах

7. **Русский язык и STT**
   - Chrome поддерживает русский язык в SpeechRecognition
   - Качество распознавания русской речи несколько ниже английской
